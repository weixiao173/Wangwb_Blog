SecureCRT的安装和使用

SecureCRT的安装和使用

一、SecureCRT工具介绍

SecureCRT是一款支持SSH(SSH1和SSH2)的终端仿真程序。通俗来说就是Windows下登录UNIX和Linux服务器主机的软件。

二、安装与破解

1、将压缩包减压到自己电脑的相应安装目录下。

2、进入到解压后的安装路径，找到SecureCRT.exe程序，右键选择“以管理员身份运行”：

3、弹出的对话框选择确定：
4、弹出对话框要求输入License（许可证），我们直接点击“下一步”：
5、弹出的新的对话框选择 “Enter License Manually” (手动输入许可证)：
6、运行破解软件 “keygen.exe”,需要注意的是杀毒软件可能会将破解机识别为木马或病毒删除，这里忽略杀毒软件的风险提示。

7、在注册机里输出Name和Company,然后点击Generate(生成)：

8、回到弹出的输入License的对话框，将注册机中的Name和Company拷贝过去，然后点击下一步：
9、将注册机中的Serial Number（序列号）和License Key拷贝过去，然后点击下一步：
10、将注册机的Issue Date（）拷贝过去，然后点击下一步，在弹出的对话框中点击完成，完成SecureCRT的安装。

三、SecureCRT的使用

1、 SecureCRT远程登录自己的阿里云服务器。首先确认几点信息。

  1. 服务器IP地址: 47.112.6.226
  2. 客户端登录账号，密码。

    2、根据实际需求按不同分类建立文件夹
    右键单击Sessions,选择New Folder

    3、建立Session(会话)
    右键单击阿里云服务器，选择 New Session,弹出如下对话框，点击下一步：

    将服务器IP，账号分别填入Hostname、UserName,点击下一步：

    弹出对话框，默认Session name为服务器IP地址，点击完成。
    双击Session name，首次远程登录，需完善用户名密码。在弹出的对话框中选择Accept&Save

    在新弹出的对话框中完善登录信息，勾选Save password，点击ok,客户端显示登录要远程服务器，可以进行之后的操作。

四、其他配置

1.配置操作日志
单击Options(选项)，选着Global options(全局选项)

然后选择General(常规)–Default Session(默认会话)–Edit Default Settings(编辑默认设置)

点击Log File，选择log file文件存储地址以及存储命名规则
G:\SecureCRT_Log%Y-%M-%D%H%Y-%M-%D_%h%m%s_%S.log
选择Append to file(追加到文件)
Upon connect:[%Y-%M-%D_%h:%m:%s]
Upon discounnect:[%Y-%M-%D_%h:%m:%s]
On each line:[%h:%m:%s]

EPC网络基本流程

EPC网络基本流程

一、概念准备

1、MM
MM（Mobility Management）移动性管理。
2、SM
SM（Session Management）会话管理。SM是与用户面数据连接有关的概念。
3、状态
4、EPS
EPS（Evolevd Packet System）演进分组系统。
EPS = UE（User Equipment，用户设备）+ LTE（4G接入网部分）+ EPC(Evolved Packet Core,演进的分组核心网)

二、EPC网络中的状态

1、EMM状态
EMM（EPS Mobility Management）演进分组系统-移动性管理
1.EMM-DEREGISTERED非注册状态
2.EMM-REGISTERED注册状态
 终端通过Attach流程注册到网络，终端进入注册态。此时终端位置被网络获知。
2、ECM状态
ECM（EPS Connection Mangagement）EPS连接管理
1.ECM-IDLE 空闲态
2.ECM-CONNECTED 连接态
注：从协议角度来说EMM和ECM是独立的。但是在EMM侧，我们谈到ECM状态时，必须是终端已经处在EMM-REGISTERED状态。终端在EMM-REGISTERED状态下，如果MME和终端之间有信令连接，终端就处于ECM-CONNECTED（连接态）。
3、RRC状态
RRC（Radio Resource Control）无线资源控制
是UE和eNodeB之间的控制平面的信令连接

三、信令连接，状态转换

1、终端UE和MME之间的包括

  1. UE和eNodeB之间的RRC连接
  2. eNodeB和MME之间的S1接口连接—NAS信令连接

在ECM-CONNECTED（连接）状态下，终端和网络侧的MME有信令连接（无线的RRC + S1-MME接口信令连接），终端和网络S-GW有用户面承载资源（无线侧的Radio Bear和S1-U接口的GTPUA承载资源）的连接。

2、4G网络的“永远在线” — always-on
终端注册到网络上之后，并不是每时每刻都具备用户面数据传送能力。
always-on的是IP Connectivity(互联网连接),并不包括无线空口承载资源。在ECM-IDLE状态下IP Connectivity存在，但是想要传送数据，还需要通过Service Request流程来建立承载。

3、终端UE和MME侧的状态转换
终端在MME侧的EMM和ECM状态的转换都是通过业务流程来转换。实际上是业务流程的需要触发了流程的转换。

  1. 终端没有业务需求就不要注册到网络上，发射和接收天线都可以关闭了——EMM-DEREGISTERED状态
  2. 终端只需开启接收天线，不需要开启发射天线——EMM-REGISTERED&ECM-IDLE状态
  3. 有持续的数据传送时网络给终端分配专有信道——ECM-CONNECTED状态
  4. 在保持信令和数据传送不影响上层业务的情况下，不连续传送就不连续传送——IDLE DRX/Active DRX/Paging特性和流程
